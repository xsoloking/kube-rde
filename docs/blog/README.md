## Contributing Blog Posts

We welcome guest posts from the community! Topics we're interested in:

- **Use Cases:** How you use KubeRDE in your organization
- **Integrations:** KubeRDE with other tools (CI/CD, IDEs, etc.)
- **Customization:** Custom workspace images or templates
- **Deployment:** Production deployment experiences
- **Comparisons:** KubeRDE vs other solutions you've tried

### Submission Process

1. **Propose a topic:** [Open a discussion](https://github.com/xsoloking/kube-rde/discussions)
2. **Get approval:** Wait for maintainer feedback
3. **Write the post:** Follow our [blog post template](#blog-post-template)
4. **Submit PR:** Create a pull request with your post
5. **Review:** Maintainers will review and suggest edits
6. **Publish:** Once approved, your post will be published

### Blog Post Template

```markdown
# Your Post Title

**Published:** Date
**Author:** Your Name
**Reading time:** X minutes

Brief introduction (1-2 paragraphs)...

## Section 1

Content...

## Section 2

Content...

## Conclusion

Summary and call to action...

---

**About the Author:**
Brief bio...

**Share this post:** [Links]
```

### Style Guide

- **Tone:** Professional but approachable
- **Length:** 1000-3000 words for technical posts
- **Code examples:** Use syntax highlighting with language tags
- **Screenshots:** Include relevant screenshots with alt text
- **Links:** Link to relevant documentation and tutorials
- **SEO:** Include relevant tags at the end

### File Naming

Use lowercase with hyphens:
```
kebab-case-title.md
```

Examples:
- `getting-started-with-kuberde.md`
- `production-deployment-guide.md`
- `custom-workspace-templates.md`
